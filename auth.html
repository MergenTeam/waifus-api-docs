Authorization and Permissions

Waifus API authorization rules and permissions to do something

As authorization process, we always use "Basic Auth".
In headers, add "Authorization" and set to Basic BASE64ENCODEDDATA and BASE64ENCODEDDATA is "email:password"s encoded phrase.

1. /auth/register REGISTER USER

Method POST
Authorization Basic Auth or None
Body id: Discord ID (Needs authorization)
or
email: User email
password: User Password
Return Success or Error

Registers user with email and password or only discord ID. Registering with email and password don't need authorization but registering with discord ID needs authorization.

2. /auth/login LOGIN

Method POST
Authorization None
Body email: User email
password: User Password
Return Success and JWT Cookie or Error

Returns JWT token to use things on website.

Permission Usage
REGISTER_USER Register user with only ID (w/o email and password)
EDIT_USER Update user data
DELETE_USER Deleting user from database
GET_USER Fetching user data
SEND_POST Uploading content
DELETE_POST Deleting content
EDIT_POST Editing content
